signed int halSpiInit ( unsigned char  port,
unsigned char  baudRate 
)

Configures the selected SPI port.

Parameters:
portmust be USCIB0
Note:
CC2530 SPI clock speed < 4MHz. SPI port configured for clock polarity of 0, clock phase of 0, and MSB first.
On MDB the RFIC SPI port is USCIB1
Modify this method for other hardware implementations.
Precondition:
SPI pins configured correctly: Clock, MOSI, MISO configured as SPI function; Chip Select configured as an output; SRDY configured as an input.
Postcondition:
SPI port is configured for RFIC communications.
Note:
immediately after calling this method you should call SPI_SS_CLEAR()

signed int halSpiWrite ( unsigned char  port,
unsigned char *  bytes,
unsigned char  numBytes 
)

Sends a message over SPI on USCI_B1. To read data out, you must write data in. To Write, set *bytes, numBytes. To Read, set *bytes only. Don't need to set numBytes because CC2530ZNP will stop when no more bytes read.

Parameters:
bytesthe data to be sent or received.
numBytesthe number of bytes to be sent. This same buffer will be overwritten with the received data.
Note:
Modify this method for other hardware implementations.
Precondition:
SPI port configured for writing
Peripheral has been initialized
Postcondition:
bytes contains received data, if any
Returns:
0 if success, -1 if incorrect port
Note:
- chip select should be controlled in the wrapping function with SPI_SS_SET() and SPI_SS_CLEAR()

Send one character out the specified port. Used in putchar().

Note, in putchar, must convert to unsigned char, e.g. int putchar(int b) { halUartCharPut(p, (unsigned char) (b & 0xFF)); return b; }

Parameters:
portmust be USCIA0 or USCIA1
thedata to send
Returns:
the data if success, else ERROR_HAL_INVALID_USCI_PORT if bad port.
